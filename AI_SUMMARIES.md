### [Link](https://powerbi.microsoft.com/en-us/blog/power-bi-2025-holiday-recap-a-decade-of-innovation-and-impact/)
Here are four technical bullet points summarizing the Power BI blog post:

* **Desktop and web data modeling parity**: Power BI has achieved core data modeling parity between Power BI Desktop and Power BI service, enabling users to create and edit semantic models directly in the Power BI service, and supporting end-to-end solution development entirely through a web browser.
* **Write Back Within Reports**: The introduction of translytical task flows (Preview) in Power BI allows users to trigger actions and write back data within reports, enabling a more seamless and interactive experience for users to turn insights into action.
* **Enhanced data modeling and version control**: Power BI now supports semantic model version history within the service, allowing creators to track changes and collaborate on data models, and automatically saving all changes made to the model.
* **Unified experience with Microsoft Fabric**: Power BI's integration with Microsoft Fabric empowers users with a unified experience for data and AI-driven innovation, enabling creators to leverage the full power of the platform to turn data into actionable insights and drive business decisions.

### [Link](https://powerbi.microsoft.com/en-us/blog/drillthrough-in-excel-now-supported-for-direct-lake-and-directquery-models/)
Here are 4 technical bullet points summarizing the Power BI blog for a professional:

* Power BI has removed the limitation on drillthrough functionality for Direct Lake and DirectQuery semantic models, allowing users to access underlying detail rows via Excel's Show Details feature.
* The MDX DRILLTHROUGH statement, previously only supported for Import models, is now supported for Direct Lake and DirectQuery models, enabling a consistent drillthrough experience across all model types.
* Drillthrough functionality respects applied filters and security roles, including Row-Level Security (RLS) and Object-Level Security (OLS) configurations, ensuring that users can only access authorized data.
* To use drillthrough, users can connect to their semantic model using Analyze in Excel, double-click a leaf-level cell in their PivotTable, and view the underlying data, with performance dependent on the underlying source system and query complexity.
